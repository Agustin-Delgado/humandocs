# Components

Everything is importable from `@human-kit/humandocs/components`. The shell pieces (`DocsShell`, `Header`, `Sidebar`, `Toc`, `ThemeToggle`) compose the page; `Demo`, `ApiReference`, `PropsTable` and `DataAttributesTable` render inside your markdown.

## Demo

`Demo` shows a live preview with a copy button and a collapsible source panel. The source comes from a `?highlight` import — the raw code plus shiki-highlighted HTML, generated at build time by the `humandocs()` Vite plugin:

<Demo source={counterSource}><Counter /></Demo>

```md
<script>
	import { Demo } from '@human-kit/humandocs/components';
	import Counter from './demos/counter.svelte';
	import counterSource from './demos/counter.svelte?highlight';
</script>

<Demo source={counterSource}><Counter /></Demo>
```

Add the ambient types to `src/app.d.ts` so `?highlight` imports are typed:

```ts
/// <reference types="@human-kit/humandocs/ambient" />
```

## DocsShell

The three-column layout: header on top, sticky sidebar on the left, content in the middle, "on this page" on the right. Regions are replaceable with snippets (`header`, `sidebar`, `toc`).

<PropsTable part="docsshell" props={sampleProps} />

`DocsShell` also forwards `title`, `badge`, `githubUrl` and `homeHref` to the default `Header`.

## ApiReference

`ApiReference` renders a full API section — one heading, props table and data-attributes table per component part — from an `api.json` generated by the [CLI](/docs/cli):

```md
<script>
	import { ApiReference } from '@human-kit/humandocs/components';
	import api from './api.json';
</script>

## API reference

<ApiReference {api} />
```

## Toc

`Toc` builds the "on this page" list from the DOM (`article h2[id], article h3[id]` by default), so it works with any content that renders headings with ids — add `rehype-slug` to get them for free. It seeds from `data.meta.headings` for SSR, then re-scans the DOM after hydration so component-rendered headings (like the ones below) are picked up too.
